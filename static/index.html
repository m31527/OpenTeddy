    Notification Credentials

    Telegram
    Inbound (Telegram → OpenTeddy)
    Lets you send messages from Telegram and have OpenTeddy run them as goals, replying with the result. Needs the server to keep running 24/7 (long-polling) — if you close the desktop app, polling stops. Strongly recommended for self-hosted server installs.
    ⚠️ Empty whitelist = inbound is OFF even if the toggle is on (we refuse to listen on an open bot — any stranger could otherwise drive the agent on your machine). Send any message to @userinfobot on Telegram to get your numeric chat_id. Toggle changes take effect on next server restart.
    SMTP / Email
    Webhook Endpoint
    External systems can POST to /webhooks/<session_id> to trigger that session. 範例： 
    curl -X POST http://localhost:8000/webhooks/YOUR_SESSION_ID \
  -H "Content-Type: application/json" \
  -H "X-OpenTeddy-Webhook-Secret: YOUR_SECRET" \
  -d '{"goal": "check for new orders and telegram me if any"}'

      The desktop app ships as a notarised bundle, so it can't live-patch itself like the web build does. Here's how to upgrade:

      1. Click Open download page below — your browser will open the GitHub Release.
      2. Download the new .dmg (ARM64 for Apple Silicon, x86_64 for Intel).
      3. Open the .dmg and drag OpenTeddy.app to Applications, overwriting the old one.
      4. Quit and relaunch OpenTeddy.

      Your sessions, memory, settings, and DB connections live in ~/Library/Application Support/OpenTeddy — replacing the .app keeps everything intact.
